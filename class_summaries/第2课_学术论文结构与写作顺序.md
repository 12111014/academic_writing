# 第2课：学术论文结构与写作顺序

## 主要内容

### 1. 论文的读者金字塔
- **Title**：约1000个读者
- **Abstract**：约200个读者（1-2段，200-400单词）
- **Introduction**：约100个读者（1-2页）
- **Related Work**：约100个读者（很多人会看这部分）
- **Method/Technique Details**：只有关心你的人或审稿人
- **Conclusion**：基本没人看

### 2. 论文结构（IMRAD模型）
- **Introduction**：为什么要做这个研究、问题是什么、假设是什么
- **Method**：怎么实现（how）、何时何地（when/where，计算机学科较少）
- **Results**：回答研究问题，展示实验结果和数据
- **Discussion**：讨论结论的含义、为什么有效、能否用到其他领域、未来工作
- **Conclusion**：扣题，与前面呼应

### 3. 漏斗式结构
- **上宽**：Introduction要画大饼，吸引读者
- **中窄**：Method和Results要实在，不能只吹不填
- **下宽**：Discussion要画更大的饼，讨论未来

### 4. 写作顺序建议
- **不要按顺序写**：Abstract最后写
- **推荐顺序**：
  1. 先写Related Work（帮助初学者理解领域）
  2. 写Method和Results
  3. 写Introduction
  4. 最后写Abstract（最精华的部分）

### 5. Related Work的重要性
- 帮助初学者梳理领域发展脉络
- 可以边读边写，总结各流派方法
- 审稿人很关注这部分，能体现作者的insight
- 如果能提出新的分类，会很有价值

### 6. 现代趋势
- **结果前置**：AI相关论文常把重要结果放在Introduction甚至Abstract
- **附录变长**：详细参数、额外分析放在附录
- **Discussion增多**：最近十年讨论部分内容越来越多
- **伦理讨论**：AI+医学等交叉领域需要讨论社会影响

### 7. 对初学者的建议
- 不要一开始就想做世界性难题
- 先解决小问题，发一系列论文积累影响力
- 现在更务实：快速发论文，而不是憋大招
- 平衡研究能力和写作包装能力

## 关键要点
1. 论文结构是固定的"八股文"，但内容可以自由发挥
2. 大部分读者只看Title和Abstract，要在这部分下功夫
3. Related Work是很好的学习工具，也是展示insight的地方
4. 写作顺序要反过来：先写技术细节，最后写Abstract
5. 图表和caption很重要，很多人只看图和表
